### Simon Willison

**[Datasette Agent](https://simonwillison.net/2026/May/21/datasette-agent/#atom-everything)** | 2026-05-21 | 5/5
Simon Willison 发布 Datasette Agent 首个版本——为 Datasette 打造的可扩展 AI 助手，将他开发三年的 LLM Python 库与 Datasette 正式整合。核心设计是对话式数据查询：用户用自然语言提问，Agent 自动生成 SQL、执行查询、返回结果。架构上采用插件系统（charts 图表可视化、sprites 沙盒代码执行、openai-imagegen 图像生成），每个能力独立为插件。演示实例跑在 Gemini 3.1 Flash-Lite 上，成本低且足以胜任 SQLite 查询。这是垂直 agent 的典型范例——不做通用 chatbot，专注数据探索场景。
**标签:** #ai-agent #datasette #plugin-architecture #data-tools

**[Gemini 3.5 Flash: more expensive, but Google plan to use it for everything](https://simonwillison.net/2026/May/19/gemini-35-flash/#atom-everything)** | 2026-05-19 | 5/5
Google I/O 上 Gemini 3.5 Flash 跳过 preview 直接 GA，但定价比前代更贵。Simon 分析了策略转向：Google 不再追求最便宜的推理成本，而是将 Flash 定位为所有产品的统一默认模型——覆盖 Gemini app、AI Mode in Search、Antigravity、Android Studio、Enterprise。这意味着"便宜快速模型"的定位正在变化，开发者需要重新评估 Flash 在成本模型中的位置。
**标签:** #gemini #llm-pricing #google-io #model-selection

### Sean Goedecke

**[Prompts are technical debt too](https://seangoedecke.com/prompts-are-technical-debt/)** | 2026-05-20 | 5/5
Sean 提出核心论点：prompt 和代码一样是技术债务。每一行 prompt 都增加系统的复杂度和维护负担——模型升级会改变 prompt 行为、上下文变化导致 prompt 失效、团队成员难以理解他人的 prompt 意图。他类比了"所有代码都是负债"的经典认知，论证 prompt 需要同样的工程纪律：版本管理、测试、review。
**标签:** #prompt-engineering #technical-debt #llm-ops #software-craft
